Секционный % в данных: `tasks[slice]`, максимум из строки 4.  
`sec_pct = sum(баллов) / max * 100`  
`avg_sec` = среднее арифметическое % по всем тестам с данными.

### Листы и порядок
- Лист `шаблон` — **пропускается**
- Листы в файле идут в **обратном** хронологическом порядке → `reversed(wb.sheetnames)`
- Лист `ЭКЗАМЕН` / `ЭКЗАМЕН (отработка)` — **школьный пробник**, не настоящий ОГЭ
  - Отображается как **обычный тест** в хронологическом порядке
  - Никакого специального выделения, никакой особой точки на графике
  - Подпись берётся из имени листа

### Группы
- Сброс нумерации (`n < prev_num` и `n <= 2`) = новая группа
- Строка «Среднее значение» = конец группы
- Нормализация имён: `name.strip().lower().split('(')[0].strip()`

## Вариант 2 — PDF для Ч/Б печати (A4, один ученик = один лист)

### Технология
1. Python генерирует **print-HTML** (только десктоп, вёрстка на `<table>`)
2. Замеряем высоту каждой страницы через **Playwright**:
```javascript
const heights = await page.evaluate(() =>
  Array.from(document.querySelectorAll('.page')).map(p => p.scrollHeight)
);
// zoom = 1062 / Math.max(...heights)  (1062px = A4 рабочая высота при 96dpi, margin 8mm)
```
3. Конвертируем через **wkhtmltopdf**:
```bash
wkhtmltopdf \
  --page-size A4 --margin-top 8mm --margin-bottom 8mm \
  --margin-left 10mm --margin-right 10mm \
  --encoding utf-8 --enable-local-file-access \
  --zoom ZOOM --dpi 96 \
  input.html output.pdf
```
При 23 учениках высота контента ~630–730px → zoom ≈ 1.46

## Обновление отчётов при добавлении новых данных

### Когда это происходит
После каждого нового теста в Excel-файл добавляется **новый лист** с результатами. Нужно сгенерировать страницы только для этого нового теста и добавить их к существующему PDF/HTML — **не пересобирать всё заново**.

---

### Алгоритм обновления

#### Шаг 1 — определить новый лист
```python
wb_old = openpyxl.load_workbook('старый_файл.xlsx', data_only=True)
wb_new = openpyxl.load_workbook('новый_файл.xlsx', data_only=True)

sheets_old = set(s for s in wb_old.sheetnames if s.strip().lower() != 'шаблон')
sheets_new = set(s for s in wb_new.sheetnames if s.strip().lower() != 'шаблон')

new_sheets = sheets_new - sheets_old  # только добавленные листы
print("Новые листы:", new_sheets)
```

#### Шаг 2 — собрать данные только по новому листу
```python
# Читаем новый лист
ws = wb_new[new_sheet_name]
rows = list(ws.iter_rows(values_only=True))

# Для каждого ученика — добавить новую запись в student_data
for row in rows[5:]:
    name = row[1]
    if not name or 'Среднее' in str(name): continue
    key = get_key(name)
    # ... парсим задания, считаем секции, итоги ...
    student_data[key].append({
        'sheet': new_sheet_name,
        'label': new_sheet_name.strip(),
        # ... остальные поля ...
    })
```

#### Шаг 3 — сгенерировать страницы только изменившихся учеников
Генерировать страницу (`make_page`) только для тех учеников, у которых появилась новая запись. Остальные ученики не трогаются.

```python
updated_keys = {get_key(row[1]) for row in rows[5:]
                if row[1] and 'Среднее' not in str(row[1]) and get_key(row[1]) in student_data}

new_pages_html = []
for g in [1,2,3]:
    for key in group_list[g]:
        if key in updated_keys:
            new_pages_html.append(make_page(key, g))
```

#### Шаг 4 — добавить страницы в существующий PDF
```python
from pypdf import PdfReader, PdfWriter
import subprocess, tempfile, os

# Генерируем HTML только с новыми страницами
tmp_html = '/tmp/new_pages.html'
with open(tmp_html, 'w', encoding='utf-8') as f:
    f.write(build_html(new_pages_html))  # тот же CSS + только новые .page

# Конвертируем в PDF
tmp_pdf = '/tmp/new_pages.pdf'
subprocess.run(['wkhtmltopdf',
    '--page-size', 'A4',
    '--margin-top', '8mm', '--margin-bottom', '8mm',
    '--margin-left', '10mm', '--margin-right', '10mm',
    '--encoding', 'utf-8', '--enable-local-file-access',
    '--zoom', str(zoom), '--dpi', '96',
    tmp_html, tmp_pdf])

# Добавляем к существующему PDF
writer = PdfWriter()
for page in PdfReader('существующий_отчёт.pdf').pages:
    writer.add_page(page)
for page in PdfReader(tmp_pdf).pages:
    writer.add_page(page)

with open('обновлённый_отчёт.pdf', 'wb') as f:
    writer.write(f)
```

### Структура анализа нового листа

При добавлении нового листа **пересчитываются** для каждого ученика:
- Средний % по тестам (`avg_pct`) — добавляется новое значение в выборку
- Средний % по каждому разделу (`avg_sec`) — аналогично
- Тренд (`trend_arrow`) — смотрит на первый и последний тест
- Рекомендации (`make_recs`) — пересчитываются на основе всей истории

Страница ученика в PDF содержит **всю его историю** (все листы от начала года), не только новый тест. Поэтому при добавлении новой страницы старая страница этого ученика **устаревает** — при желании её можно удалить из PDF или заменить.

## Соответствие листа варианту сборника

Для будущих обновлений действует жёсткое правило:

- если в названии листа есть `1 TEST`, это соответствует **варианту 1** из сборника
- если в названии листа есть `2 TEST`, это соответствует **варианту 2** из сборника
- ...
- если в названии листа есть `8 TEST`, это соответствует **варианту 8** из сборника

Общее правило:

```python
N TEST  ->  вариант N из сборника
```

Это соответствие используется для анализа ошибок:

- сравнивать результаты ученика с заданиями именно этого варианта из сборника
- выводить проблемные места по чтению, грамматике и словообразованию именно по этому варианту
- если лист называется `8 TEST ...`, ошибки и рекомендации строятся по **8 варианту** из сборника

Если номер варианта удалось извлечь из имени листа, анализ должен опираться не только на общий % по разделу, но и на содержание соответствующего варианта сборника.

Это правило действует для всех вариантов из этого сборника:
- `1 TEST ...` -> вариант `1`
- `2 TEST ...` -> вариант `2`
- `3 TEST ...` -> вариант `3`
- ...
- `15 TEST ...` -> вариант `15`

То же правило должно автоматически применяться и в будущем:
- если в новых Excel-таблицах появятся листы `N TEST ...`, их нужно сразу считать вариантом `N` из этого же сборника
- при каждом новом импорте нужно делать переанализ не только новых листов, но и уже прорешанных вариантов, чтобы сводка ошибок по чтению, грамматике и словообразованию оставалась актуальной

| 30 | Прилагательные | -ous, -ful, -al, -ive, -able/-ible | FAME→famous, SCIENCE→scientific, NOISE→noisy, WONDER→wonderful |
| 31 | Существительные (деятель/результат) | -er/-or, -ist, -ness, -ion | WIN→winner, EXPLORE→explorer, SUCCESS→successful |
| 32 | Прилагательные | -able, -ful, -ous, -ive, -ing/-ed | FASHION→fashionable, NERVE→nervous, EXCITE→exciting |

| 34 | Прилагательные с отрицательным префиксом | un-, im-, in-, dis-, ir- | FAIR→unfair, USE→useless, LUCK→unlucky, USUAL→unusual |

**Вывод:** задания 29–34 проверяют:
1. **Суффиксы существительных** (-tion, -ance, -er, -ment) — задания 29, 31
2. **Суффиксы прилагательных** (-ous, -ful, -al, -ive, -able) — задания 30, 32, 33
3. **Отрицательные префиксы** (un-, im-, in-, dis-) — задания 33, 34 (частая ловушка)

### Как использовать карту в рекомендациях

Вместо абстрактного «Грамматика 65%» в блоке рекомендаций — конкретная тема:

```python
# Расширенные рекомендации по заданиям (если известны баллы по заданиям)

    26: 'Отрицательные формы / Числительные',

    29: 'Словообраз.: существительные (-tion, -ance)',
```
